
# Multi-Agent AI System

This project is a `Multi-Agent` retrieval system on the top of KPN's publicly available data. It exposes a `FastAPI` API that accepts a user query, routes that query to an agentic system built using LangGraph with two specialized agents:

- `annual_report_agent`: answers from the KPN 2024 annual report.
- `news_agent`: answers from scraped KPN news articles. For demo, only 5 news articles are scraped, but can be extended further.

Under the hood, both agents use `AzureOpenAI` for embeddings and generation, and `FAISS` for local vector storage.

🎬 **Demo Video:** [Watch here](https://drive.google.com/file/d/1QCfOHBUSbSjt8G4uwfEtL3dKc9VSFdNP/view?usp=sharing)

## What The Project Does

The runtime flow is:

1. A client sends a question to the FastAPI `/query` endpoint.
2. The LangGraph router identifies the query `intent` and routes it to either `annual_report` or `news` agent.
3. The selected agent performs similarity search on its FAISS index.
4. The retrieved text is passed to the LLM with a constrained prompt.
5. The API returns the final answer, along with the `source`.

Typical examples:

- "What is KPN purpose and mission?" -> routed to the annual report agent
- "What is KPN Campus?" -> routed to the news agent

## Architecture Summary

Core pieces:

- FastAPI API layer in `api/app.py`
- LangGraph agent `src/graph/`
- Azure OpenAI model and embedding setup in `src/models/llm.py`
- Retrieval agents in `src/tools/`
- Data preparation utilities in `src/utils/`
- Local FAISS vectorstore in `src/vectorstores/faiss.py`

## Prerequisites

Before running the project, make sure you have:

- Python 3.12
- `uv` installed
- Azure OpenAI access and deployment configuration
- Docker installed if you want to run the containerized version

## Environment Variables

The app loads environment variables from a local `.env` file. At minimum, define:

```env
AZURE_OPENAI_API_KEY=your-key
AZURE_OPENAI_API_VERSION=your-api-version
AZURE_OPENAI_ENDPOINT=https://your-resource.openai.azure.com/
AZURE_OPENAI_GPT_DEPLOYMENT=your-chat-deployment-name
AZURE_OPENAI_EMBEDDING_DEPLOYMENT=your-embedding-deployment-name
```

## Setup With uv

From the project root, create the environment and install dependencies:

```bash
cd path_to_project_root
uv sync
```

## Running `run_docker.sh`

The repository includes a helper script at `run_docker.sh` to start the Docker container without typing the full `docker run` command each time.

First, build the image from the project root:

```bash
docker build -t kpn-api .
```

Run it with Bash:

```bash
bash run_docker.sh
```

Or, if it is executable in your shell:

```bash
./run_docker.sh
```

Once running, the API is typically available at:

```text
http://127.0.0.1:8000
```

Useful endpoints:

- `GET /health`
- `POST /query`

## Running Locally

Start the FastAPI app with Uvicorn:

```bash
uv run uvicorn api.app:api --reload
```

## Data Flow And Storage

The project uses a Medellian Architecture, a layered data design pattern which improves data quality incrementally while maintaining scalability and governance. Below are the layers:

- `bronze`: raw source inputs
- `silver`: processed intermediate artifacts (JSON, JSONL, NDJSON etc.)
- `gold`: ready-to-query vector indexes

Current behavior:

- The annual report agent loads preprocessed JSONL from `data/silver` when available.
- If the parsed file is missing, it attempts to parse the raw PDF from `data/bronze` using Docling.
- Both agents create or reuse FAISS indexes under `data/gold`.

## Folder Guide

### Root

- `pyproject.toml`: project metadata and Python dependencies
- `uv.lock`: lockfile generated by `uv`
- `Dockerfile`: container definition for running the API with Python 3.12
- `run_docker.sh`: Bash helper script that starts the Docker container
- `.python-version`: local Python version marker
- `.env`: local environment variables for Azure OpenAI and any other secrets

### `api/`

- `app.py`: FastAPI application entrypoint exposing `/health` and `/query`

### `src/graph/`

- `graph.py`: builds the LangGraph workflow and compiles the app graph
- `router.py`: routing node that decides whether a query should go to `annual_report` or `news`
- `nodes.py`: node handlers that call the specialized agents
- `state.py`: typed graph state definition

### `src/models/`

- `llm.py`: creates the `AzureOpenAI` model and embedding model

### `src/tools/`

- `annual_report_agent.py`: retrieval and answer generation for annual report questions
- `news_agent.py`: retrieval and answer generation for KPN news questions

### `src/utils/`

- `paths.py`: central filesystem path definitions for `data/bronze`, `data/silver`, and `data/gold`
- `pdf_parser.py`: loads existing parsed report data or parses a PDF into documents
- `web_scraper.py`: loads URLs and converts fetched HTML pages into LangChain documents
- `common.py`: JSONL save/load helpers for document persistence

### `src/vectorstores/`

- `faiss.py`: creates, loads, and saves FAISS vector stores locally

### `src/logging/`

- `logger.py`: configures console and file logging

### `data/`

- `data/bronze/`: raw input assets, including `urls.txt` and expected source documents
- `data/silver/`: processed intermediate outputs such as parsed JSONL files
- `data/gold/`: persisted FAISS indexes for fast query-time retrieval

### `logs/`

- application logs, including `logs/app.log`

## Troubleshooting

If the app fails on startup:

- verify the `.env` values are present and valid
- run `uv sync` again to ensure dependencies are installed
- confirm the expected data files exist under `data/bronze`, `data/silver`, and `data/gold`
- inspect `logs/app.log` for errors

If Docker starts but the API is unreachable:

- verify the container is running
- confirm the command exposes port `8000`
- ensure nothing else on your machine is already using port `8000`
